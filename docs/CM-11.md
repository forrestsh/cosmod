# CM-11: Publish Cosmic Grid Particles demo with help guide

## Summary

As the site manager, I want to publish the **Cosmic Grid Particles** mobile-friendly demo as a discoverable application within the Cosmod site (alongside the existing simulation cards and Crystal Garden), so visitors can explore five canonical FCC 12-direction particle-physics scenarios — from a BDEF baryon's period-4 heartbeat to a full 30-tick baryon/anti-baryon collision — with an accompanying help guide for reading what they're seeing.

## Background

`cosmic_grid_particles_mobile.jsx` is a React + Three.js component that visualises particle dynamics on the FCC 12-direction lattice, reading state as a 12-bit vector per cell, with three rule families (`rt_FCC`, `rt_P1`, `rt_6cycle`) supporting five scripted scenarios:

1. **BDEF baryon** — 4-bit bound state, period-4 heartbeat (compact → split → compact), drift speed √2/4 ≈ 0.354c.
2. **B + b pair annihilation** — minimal anti-particle reaction; rt[130]↔rt[65] swap turns B+b into A+a, igniting a gun.
3. **Baryon / anti-baryon collision** — full 30-tick script: Compton radiation at t=13, collision core at t=16, annihilation at t=19, gun ignition at t=20.
4. **Light-speed gun (rt_FCC)** — the original single-A-bit gun as a baseline contrast.
5. **6-cycle colour wheel F+f → gun** — full Aa↔Bb↔Cc↔Dd↔Ee↔Ff↔Aa swap chain in action.

The companion `cosmic_grid_particles_demo_guide.md` explains the geometry (12 directions × colour family), the three rule tables, the controls (drag/pinch/slider/play/trails/grid), per-scenario tick-by-tick scripts, and the framework-level claims each scenario evidences.

The deliverables already exist in `~/Downloads/`:

- `cosmic_grid_particles_mobile.jsx` — the React component.
- `cosmic_grid_particles_demo_guide.md` — the help guide (Chinese with technical English terms).

This is similar in spirit to CM-10 (Crystal Garden) — a single experience plus a manual — but unlike Crystal Garden (which was a single-file HTML served statically) this demo is a React component that lives inside the existing SPA and follows the same routing pattern as the other simulation cards (Hex, Toroidal, 4x4, Magic Search).

## User-facing behavior

1. A new card on the homepage links to the Cosmic Grid Particles demo, between the existing simulation cards and the Crystal Garden / Book cards.
2. Clicking the card navigates to the demo's React route (e.g. `/particles`); the existing `← Home` chrome from `ModelShell` is reused so users can return.
3. The demo loads with the **collision** scenario selected by default (the most visually rich script) and shows:
   - A scrollable scenario-chip row with five scenarios (`BDEF 重子`, `B + b 对湮灭`, `重子-反重子对撞`, `Light-speed gun`, `色轮 F+f → gun`).
   - A 3D viewport with FCC ghost lattice (toggleable), origin ring, particle spheres coloured by active-bit blend, and per-bit direction arrows.
   - HUD top-left: `t NN`, `cells N · bits N`.
   - HUD top-right: control hints (`drag · rotate`, `pinch · zoom`).
   - Bottom caption: amber per-tick annotation when present, otherwise the scenario description.
   - Bottom controls: frame slider, ⟲ / ◀ / ▶ play / ▶ next, **TRAILS** toggle, **GRID** toggle.
4. Controls match the guide:
   - **Desktop:** drag to rotate, scroll to zoom, click chips to switch scenarios, slider to scrub frames.
   - **Mobile:** one-finger drag to rotate, two-finger pinch to zoom, swipe the chip row.
   - Auto-play advances one frame every 700 ms and stops at the last frame.
   - **TRAILS** renders the previous 6 frames as fading dots; **GRID** shows the FCC ghost sublattice.
5. A **Help** (or **Manual**) link opens the `cosmic_grid_particles_demo_guide.md` content rendered as styled HTML — same pattern as the Crystal Garden manual page.

## Site-integration expectations

1. Place the component at the repo root as `cosmic-grid-particles.jsx` (matching the existing kebab-case pattern of `cosmic-grid-4x4.jsx`, `cosmic-grid-toroidal.jsx`, `hex-cosmic-grid.jsx`, `cosmic-magic-search.jsx`).
2. Register a React Router route in `src/App.jsx` (e.g. `/particles`) wrapped in the existing `ModelShell` so the floating `← Home` link is provided automatically.
3. Add a homepage card in `src/HomePage.jsx` matching the existing card pattern (uppercase title, monospace description, hover state). Title and short description should call out the headline scenarios — e.g. "5 scripted scenarios on the FCC 12-direction lattice — baryon heartbeat, pair annihilation, full collision, light-speed gun".
4. Add the **`three`** package as a dependency (the component imports `import * as THREE from 'three'`) and verify the existing build (`vite build`) bundles it cleanly.
5. Publish the help guide so users can read it from inside the demo:
   - Either render `cosmic_grid_particles_demo_guide.md` via marked.js at a static path (`public/cosmic-grid-particles/manual.html`, mirroring `public/crystal-garden/manual.html`), and link to it from the in-app Help button — **or** an in-app help drawer if simpler.
   - If choosing the static-manual route, update `vercel.json` so `/cosmic-grid-particles/` bypasses the SPA catch-all alongside `/assets/`, `/books/`, and `/crystal-garden/`.
6. Source artifacts (`cosmic_grid_particles_mobile.jsx` original and `cosmic_grid_particles_demo_guide.md`) should be checked into the repo for traceability.

## Acceptance criteria

1. A new card titled **Cosmic Grid Particles** (or equivalent) appears on the homepage and links to the demo route.
2. The demo loads with all five scenarios selectable; switching scenarios resets the frame to 0 and pauses playback.
3. The 3D viewport renders particle spheres with per-direction arrows and the colour blend specified in the guide; popcount ≥ 4 cells render at 1.4× scale.
4. Frame slider, play/pause, single-step, reset, **TRAILS**, and **GRID** controls all behave as described in the guide.
5. Per-tick annotations appear in amber for the documented frames (e.g. collision t=0/12/13/16/19/20); other frames show the scenario description in cool grey.
6. Mobile interactions work: single-finger drag rotates, pinch zooms, chip row scrolls horizontally with smooth touch.
7. The help guide is reachable from the demo (clearly labelled link or button) and renders the markdown with proper headings, tables, code blocks, and lists.
8. Production deploy on Vercel loads the demo without console errors that block rendering, and the React Router catch-all does not intercept the manual-page path (if a static manual is used).
9. `three` is in `package.json` and the production build succeeds.

## Representative content highlights (informational)

These are the headline beats a visitor should be able to find and watch:

- **BDEF baryon** — at t=0 / 4 / 8 the four bits collapse back to a single cell; mid-cycle they fan out to four cells. Caption explains the period-4 heartbeat and the √2/4c drift.
- **B + b annihilation** — at t=1 the heads meet at the origin and the rt[B+b]=A+a swap fires; t=3 lights the gun.
- **Baryon collision** — the most cinematic: Compton radiation (t=13), collision core (t=16), annihilation (t=19), gun ignition + wires (t=20).
- **Light-speed gun** — bits(t) = t+1 along the (0,+y,+z) corridor; pure baseline.
- **F + f → gun** — colour-wheel closure rt[Ff]=Aa, then standard gun.

## Out of scope (unless amended)

- Authoring new scenarios or rules in-browser.
- Editing rule tables; switching rule families is implicit per scenario only.
- Saving / exporting frames or rule sequences (no JSON export, no STL).
- Translating the help guide into English (the source is Chinese with English physics terms; a future ticket can localise).
- Performance work for very large scenarios — the five scripted scenarios run in well under 40 frames each.
